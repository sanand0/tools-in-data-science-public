# Tools in Data Science
## Live session 1: orientation

- What the course is trying to make you fluent at
- How graded work, projects, and live support fit together
- How to learn with AI without outsourcing your thinking

notes: Set the tone early: the course rewards working systems, repeatable commands, and clear evidence.
---
## The course loop

1. Read the topic note
2. Try the command or workflow
3. Break it on a small example
4. Ask AI for diagnosis
5. Verify with your own run
6. Keep a short trail of evidence
---
## What counts as evidence?

- Terminal command and output
- Screenshot of the app/page/result
- URL of a deployed artifact
- GitHub commit or file path
- A short explanation of why the result answers the question
---
## AI is part of the syllabus

- Ask for explanations, variations, and tests
- Ask twice before escalating to humans
- Never trust an answer until it runs
- Keep prompts narrow and results inspectable
---

## A good student workflow

```bash
mkdir tds-practice
cd tds-practice
uv init
uv run python -c "print('ready')"
git init
git status
```

Keep the project small enough that you can recreate it.
